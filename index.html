GitHub Pages Tutorial

Guía interactiva de publicación rápida

Hecho para IA Arena (Agente)

¡Listo para Publicar!

Tu Agente de IA ya se ha encargado de estructurar el código de tu sitio. Sigue esta guía visual paso a paso para publicarlo gratis en internet.

Progreso del Despliegue 0%

Requisitos Previos

  • Una cuenta activa en GitHub.
  • Código del proyecto web listo.
Paso 1

Crear un Nuevo Repositorio en GitHub

  1. 1 Inicia sesión en tu cuenta en github.com .
  2. 2 En la página principal o en la pestaña "Repositories", busca y haz clic en el botón verde New (Nuevo).
  3. 3 Asigna un nombre para tu proyecto en el campo Repository name (ej: mi-sitio-web).
  4. 4 Asegúrate de marcar el repositorio como Public (es indispensable para usar la opción gratuita de Pages).
  5. 5 Deja las demás casillas opcionales desmarcadas y haz clic en el botón Create repository.
Paso 2

Subir los archivos de tu proyecto

  1. 1 En la pantalla que se muestra, busca y haz clic en el enlace que dice "uploading an existing file".
  2. 2 Arrastra y suelta todos los archivos de tu web dentro de la zona de carga del navegador.
  3. 3 En la parte inferior de la página, en el formulario de confirmación, escribe un título breve para tus cambios (ej: Subida inicial).
  4. 4 Presiona el botón verde Commit changes y espera a que GitHub procese los archivos.
Paso 3

Activar el Despliegue de Pages

  1. 1 En la barra superior de opciones de tu repositorio, haz clic en la pestaña Settings (Configuración).
  2. 2 En el panel de navegación izquierdo, dentro de la sección "Code and automation", haz clic en Pages.
  3. 3 Bajo la sección *Build and deployment*, cambia el selector de la rama (Branch) que por defecto está en None por **main** (o master).
  4. 4 Haz clic en el botón de guardar Save. ¡Tu sitio se empezará a generar automáticamente!
Paso 4

Actualizar tu web y subir cambios

  1. 1 Accede a tu repositorio en GitHub y haz clic en el botón verde Add file → Upload files.
  2. 2 Arrastra y suelta los archivos que hayas modificado (o las nuevas versiones).
  3. 3 En la parte inferior, escribe un mensaje descriptivo de los cambios (ej: Actualización del diseño).
  4. 4 Haz clic en Commit changes. Los cambios se subirán automáticamente a la rama main.
  5. 5 GitHub Pages se actualizará automáticamente en 1-5 minutos. ¡Tu web mostrará los cambios!

¿Dónde estará visible tu página?

Cuando el despliegue finalice, podrás acceder a tu sitio web utilizando este enlace personalizado:

https://tu-usuario.github.io/tu-repositorio/

Nota de despliegue: El proceso inicial toma habitualmente entre 1 y 5 minutos.
